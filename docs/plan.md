# 강의용 슬라이드 사이트 구축

## 사이트 기능 요구사항

- 다양한 강의 자료를 슬라이드 형태로 보여주는 플랫폼
- MKDocs, Reveal.js 기반이 괜찮을 것으로 예상함.
- 윈도우 개인 PC에서 구축 테스트 후 Github를 통해 rocky linux9 서버에 배포 호스팅
- pm2 ecoconfig 로 여러 웹서비스 현재 구동 중이며, pm2를 통해 관리할 예정임.
- 강의 슬라이드에 필요에 따라 각종 응용 목업이 데모 형식으로 보여질 수 있는 구조.
- 윈도우에서는 구축하여 localhost에서 테스트 하지만, 리눅스 배포 서버는 edadev.ednc.co.kr URL과 특정 포트(예:8900)를 통해 서비스 가능하도록 할 예정.  
- 강의가 여러 개 등록될 경우 url을 어떻게 할당할지 고려되어야 함. (예: edadev.ednc.co.kr/mb/lec01, edadev.ednc.co.kr/mb/lec02 등)
- 강의 슬라이드 문장들은 개조식, 입니다,합니다 같은 서술형 문장은 지양하고, 핵심 키워드 위주로 구성.
- 전체 사이트 테마는 기술적인 내용을 전달하는 것이 목적이므로, 편안한 느낌이 나는 테마를 선택할 예정이며, 너무 어둡지 않고 테마도 쉽게 바꿀 수 있는 구조였으면 좋겠어. 강의 마다 다른 테마를 선택할 수 있다면 더 좋을 것 같음.


## Arty A7 보드를 활용하여 MicroBlaze 소프트 프로세서와 사용자 정의 하드웨어(Custom Logic)를 연동하는 실습

- 초보자가 하드웨어 설계부터 소프트웨어 제어까지 단계별로 체감할 수 있도록 구성한 4가지 예제와 교육 계획안

### 예제 1: MicroBlaze & 기본 GPIO 제어

- 내용: MicroBlaze 최소 시스템을 구성하고, 보드의 LED와 Switch를 AXI GPIO IP로 제어.
- 학습 목표: Vivado의 Block Design 흐름과 Vitis(SDK)를 통한 C 언어 코딩 기초 습득.

### 예제 2: 타이머 인터럽트를 활용한 RGB LED 제어

- 내용: AXI Timer와 Interrupt Controller를 추가하여 주기적으로 RGB LED의 색상을 변경. 
- 학습 목표: 하드웨어 인터럽트 개념 이해 및 다중 IP 연동 방법 숙지.

### 예제 3: 고속 데이터 전송을 위한 AXI-Stream & FIFO 실습

- 배경: AXI-Lite는 CPU가 레지스터를 하나씩 읽고 쓰는 방식이라 느림. 대량의 데이터를 주고받을 때는 AXI-Stream 사용.
- 내용: MicroBlaze에서 생성한 데이터를 FIFO(First-In-First-Out) 메모리를 거쳐 하드웨어 로직으로 쏘고, 다시 돌아오는 루프백(Loopback) 시스템을 구축.
- 학습 목표: "주소(Address)" 없이 흐르는 데이터 스트림 개념 이해 및 버퍼링(FIFO)의 필요성 습득.
- Arty A7 활용: 보드의 버튼을 누르면 대량의 난수가 생성되어 UART 터미널로 결과값이 쏟아지는 구조.

### 예제 4: 외부 메모리(DDR3)와 Direct Memory Access (DMA) 연동

- 배경: CPU를 거치지 않고 하드웨어가 직접 메모리에 접근하는 DMA는 고성능 SoC의 필수 요소.
- 내용: Arty A7에 탑재된 256MB DDR3L 메모리를 활용합니다. MicroBlaze가 메모리에 데이터를 써두면, AXI DMA가 이 데이터를 읽어와 사용자 로직(가속기)에 전달하고 결과를 다시 메모리에 저장.
- 학습 목표: CPU 점유율을 낮추는 DMA 제어 및 외부 메모리 맵 관리 방법 습득.
Arty A7 활용: 보드의 DDR3 메모리 컨트롤러(MIG IP)를 실제로 연동해 보는 하이엔드 실습.

### 요약: 1~4단계 교육 커리큘럼 (최종)

| 단계 | 제목 | 핵심 키워드 | 학습 경험 |
| :---: | :--- | :--- | :--- |
| 1 | MicroBlaze | AXI-GPIO | CPU를 올리고 단순 On/Off 제어 |
| 2 | Interrupt 제어 | Interrupt Controller | 하드웨어 이벤트에 실시간 반응 |
| 3 | 데이터 스트림 처리 | AXI-Stream / FIFO | 주소 없이 빠르게 흐르는 데이터 제어 |
| 4 | 메모리 직접 접근(DMA) | DDR3 / AXI DMA | 대용량 데이터 전송 및 외부 메모리 활용 |
