# 강의용 슬라이드 사이트 구현 계획서

## 1. 개요
본 문서는 `docs/plan.md`에 명시된 요구사항을 바탕으로, MicroBlaze 실습 교육을 위한 슬라이드 기반 웹 사이트를 성공적으로 구축하고 배포하기 위한 전문적인 기술 구현 계획입니다. 사용자의 피드백을 반영하여 프레임워크와 배포 방식을 확정하였습니다.

## 2. 확정된 주요 아키텍처 (Architecture Decisions)

### 2.1 메인 프레임워크: Next.js + Reveal.js
복잡하고 인터랙티브한 대화형 데모 목업을 슬라이드 내에 원활하게 삽입하기 위해 **Next.js + Reveal.js (React 기반)** 환경을 사용합니다. 마크다운 기반의 작성 편리함과 모던 프론트엔드 React 컴포넌트 확장을 모두 만족하는 구조로 진행합니다.

### 2.2 다중 강의 URL 라우팅 및 서버 환경
Nginx 등의 추가 프록시 웹 서버 없이, **Node.js(Express) 커스텀 웹 서버**를 작성하여 다중 강의(예: `/mb/lec01`, `/mb/lec02`) 정적 빌드 파일들을 라우팅 및 서빙하도록 구축합니다.
전체 서버는 **PM2**를 사용하여 리눅스 서버(Rocky 9)에서 안정적으로 구동됩니다.

### 2.3 테마 및 디자인 커스터마이징
**민트(Mint)** 또는 **다크 그린(Dark Green)** 계열을 대표 테마 색상으로 설정합니다. 가독성이 높고 눈이 편안한 모던 테마(다크/라이트 모드 지원)로 구성하며, 강의마다 색상 테마를 바꿀 수 있도록 CSS Variables를 전역으로 설계합니다.

### 2.4 모듈화 및 URL 라우팅 전략 (Next.js App Router)
하나의 파일에 모든 기능이 밀집되는 것을 방지하기 위해 Next.js의 App Router 아키텍처를 적극 활용합니다.
- **포털 진입점 (`/`)**: 메인 대시보드 (`src/app/page.tsx`)에서 개별 교육 과정으로 진입할 수 있는 네비게이션 역할을 수행합니다.
- **개별 과정 페이지 (`/mb`, `/zynq` 등)**: 과정별로 폴더 기반 라우팅을 분리하여 각 슬라이드를 독립적으로 렌더링합니다 (`src/app/mb/page.tsx`).
- **UI 컴포넌트 분리 (`src/components/`)**: `RevealWrapper`, `ImagePlaceholder`, `WorkflowDiagram` 등 복잡한 UI와 Animation, 상태(State) 로직을 갖는 모듈들은 공통 컴포넌트 폴더로 추출하여 재사용성을 극대화합니다.

## 3. 세부 구현 단계 (Action Plan)

### Phase 1: 기반 아키텍처 및 로컬 환경 템플릿 구축 (Windows)
1. 프론트엔드 프로젝트(Next.js 기반)와 Express 라우팅 서버를 초기화합니다.
2. 민트/다크 그린 기반 글로벌 디자인 시스템 설정 및 컴포넌트화, 강의별 스타일 메타데이터 적용 구조를 설계합니다.
3. 로컬 윈도우 환경에서 `npm run dev` 스크립트 및 컴포넌트 동작을 테스트할 수 있는 환경을 세팅합니다.

### Phase 2: 슬라이드 스켈레톤 구성 및 목업 연동 테스트
1. `docs/plan.md`의 커리큘럼(예제 1~4)에 맞춰 뼈대(Skeleton) 마크다운 슬라이드 파일들을 분리하여 생성합니다.
2. 서술형 문장 지양 원칙에 맞게 개조식 문장에 최적화된 프레젠테이션 타이포그래피(가독성 레벨링) 및 그리드 레이아웃을 설정합니다.
3. 응용 목업 컴포넌트(React 컴포넌트)를 슬라이드 중간에 삽입하여 정상 렌더링 및 인터랙션이 구동되는지 테스트를 진행합니다.

### Phase 2.5: 코드 모듈화 및 App Router 분리 리팩토링
1. 메인 라우터(`src/app/page.tsx`)를 대시보드 포털로 전환합니다.
2. 컴포넌트 폴더(`src/components/`)를 생성하여 재사용 가능한 UI 모듈을 추출합니다 (`ImagePlaceholder.tsx`, `WorkflowDiagram.tsx`, `RevealWrapper.tsx`).
3. MicroBlaze 강의 코드를 전용 라우터(`src/app/mb/page.tsx`)로 완벽히 분리 및 이관합니다.

### Phase 3: Linux 서버 배포 자동화 및 PM2 연동 (Rocky 9 배포 준비)
1. `.env` 파일을 도입하여 포트(8900)와 호스트 주소를 동적으로 관리하도록 구성합니다. 하드코딩된 값을 배제합니다.
2. `ecosystem.config.js` 스크립트를 작성하여 `.env` 환경 변수를 로드하고 Node.js Express 서버를 PM2로 구동하도록 설정을 마칩니다.
3. 배포 및 관리는 제공해주신 Git 저장소(`https://github.com/ahe24/edu_mb.git`)를 활용합니다.
4. 손쉬운 서버 배포를 위해 인수인계 및 운영 가이드 문서(`README.md`)에 `git clone` 이후부터 라이브 서비스 구동까지의 명령어 스텝을 정리합니다.

### Phase 4: 최종 테스트 및 최적화
1. 윈도우 로컬에서 실제 빌드 결과물(Static HTML) 및 PM2 라우팅 동작 최종 테스트 (Node.js Express 단일 진입점에서 여러 강의 확인).
2. 브라우저 크기/비율에 따른 반응형 슬라이드 레이아웃(디스플레이 비율 등) 최적화.

## 4. 검증 계획 (Verification Plan)
- **로컬 구동 검증 (Manual):** 전체 정적 빌드 완료 후 `pm2 start ecosystem.config.js`를 윈도우 로컬에서 실행하여 `http://localhost:8900/mb/lec01` 포맷의 라우팅 진입이 정상 동작하는지 육안으로 확인합니다.
- **데모 컴포넌트 검증 (Manual):** 임의의 대화형 더미 목업(React) 컴포넌트를 슬라이드 페이지에 삽입하여, 페이지 전환 및 이벤트 발생 시 충돌 없이 정상 구동됨을 검증합니다.
